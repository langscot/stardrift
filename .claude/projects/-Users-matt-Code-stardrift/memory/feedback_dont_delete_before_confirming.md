---
name: Don't delete before confirming replacements
description: Never delete existing generated assets before the user confirms the new version looks good
type: feedback
---

Don't delete existing generated assets (images, etc.) before the user has confirmed the replacement looks good.

**Why:** User lost previously generated emoji images because I deleted them before verifying the new style was acceptable.

**How to apply:** When regenerating assets, save new versions alongside the old ones (e.g. in a temp dir or with a suffix) until the user confirms, then swap.
